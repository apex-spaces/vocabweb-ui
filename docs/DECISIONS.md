# DECISIONS.md - 设计决策记录

## D-001：产品定位
- **日期**：2026-02-07
- **决策**：Web优先 + 适度游戏化
- **理由**：主流背单词产品都是App优先，Web端有差异化空间；游戏化提升留存

## D-002：目标用户
- **日期**：2026-02-07
- **决策**：先只做英语，面向雅思/托福/考研/GRE考试群体
- **理由**：考试群体需求明确，付费意愿强

## D-003：前端技术栈
- **日期**：2026-02-07
- **决策**：Next.js 14 + Tailwind CSS + shadcn/ui
- **理由**：SSR/SSG兼备，SEO友好，组件质量高

## D-004：后端技术栈
- **日期**：2026-02-07
- **决策**：Go，前后端分离，各自部署Cloud Run
- **理由**：老板规范——后端默认用Go；性能好，适合API服务

## D-005：认证方案
- **日期**：2026-02-07
- **决策**：Google Identity Platform
- **理由**：GCP原生，gcloud可启用，不需要额外申请

## D-006：数据库
- **日期**：2026-02-07
- **决策**：Cloud SQL (PostgreSQL 15)
- **理由**：GCP托管，自动备份，统一管理

## D-007：图片识别方案
- **日期**：2026-02-07
- **决策**：两步方案——Cloud Vision API (OCR) + Vertex AI Gemini (生词分析)
- **理由**：各司其职精度最高，全在GCP内统一计费

## D-008：部署规范
- **日期**：2026-02-07
- **决策**：项目 openclaw-lytzju / 区域 asia-east2（香港）
- **理由**：统一部署项目，香港区域延迟低

## D-009：游戏化功能
- **日期**：2026-02-07
- **决策**：连续打卡 + 成就徽章 + 等级系统 + 排行榜 + 学习目标挑战
- **理由**：提升用户留存和学习动力

## D-010：考试功能
- **日期**：2026-02-07
- **决策**：内置考试词库 + 考试模式 + 词汇量测试 + 考频标注 + 备考计划
- **理由**：面向考试群体的核心需求

## D-011：核心原则
- **日期**：2026-02-07
- **决策**：尽可能使用Google Cloud基础设施，统一运维
- **理由**：一个项目一个账单，运维最简
